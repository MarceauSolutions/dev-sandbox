3. **Different Time Requirements**:
   - Fitness: Mostly automated (X posts, MCP tools)
   - Elder Tech: Requires focused dev time
4. **Shared Infrastructure**: Both use Claude API, Twilio, Gmail integrations

### Balance Rule
- **Fitness**: 40% of dev time (already built, now marketing/iteration)
- **Elder Tech**: 60% of dev time (new build, MVP priority)

### Switch Triggers
- If Fitness revenue > $500/mo: Shift more time to Fitness
- If Elder Tech beta user commits: Shift to finish MVP
- If X engagement < 1%: Reduce Fitness posting, review strategy

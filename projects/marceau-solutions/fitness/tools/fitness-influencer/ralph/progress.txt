**Filler Categories**:
| Category | Examples | Base Confidence |
|----------|----------|-----------------|
| HESITATION | um, uh, er, ah | 0.85-0.95 |
| DISCOURSE_MARKER | like, so, well | 0.60-0.70 |
| HEDGE | basically, actually, literally, just | 0.50-0.80 |

**Context-Aware Detection**:
```python
# Examples of context patterns that modify confidence
"like":
  - "^like," → +0.9 (filler usage)
  - "it's like" → -0.5 (comparison, keep)
  - "looks like" → -0.6 (comparison, keep)

"so":
  - "^so um" → +0.9 (definitely filler)
  - "so that" → -0.7 (connector, keep)
```

**Sensitivity Presets**:
| Preset | Threshold | Categories | Cluster Gap |
|--------|-----------|------------|-------------|
| aggressive | 0.5 | All | 0.8s |
| moderate | 0.7 | All | 0.5s |
| conservative | 0.85 | HESITATION, TAG only | 0.3s |

**API Response**:

```json
    "filler_percentage": 4.5,

  },
  "summary": { "hesitations": 5, "discourse_markers": 3, ... }
}
```

**FFmpeg Processing Pipeline**:
```
Input Video
    │
    ▼
[Detect Filler Segments: 0.5s-0.8s, 2.3s-2.5s, 5.0s-5.3s]
    │
    ▼
[Calculate Keep Segments: 0-0.5s, 0.8s-2.3s, 2.5s-5.0s, 5.3s-end]
    │
    ▼
[FFmpeg filter_complex]
    ├── [0:v]trim=0:0.5,setpts=PTS-STARTPTS[v0]
    ├── [0:v]trim=0.8:2.3,setpts=PTS-STARTPTS[v1]
    ├── ... (video segments)
    ├── [0:a]atrim=0:0.5,asetpts=PTS-STARTPTS[a0]
    ├── [0:a]atrim=0.8:2.3,asetpts=PTS-STARTPTS[a1]
    ├── ... (audio segments)
    └── concat=n=N:v=1:a=1[outv][outa]
    │
    ▼
Output: Clean video with smooth transitions
```

**Audio Crossfade Implementation**:
- 50ms fade in/out at each cut point
- Prevents audio pops and clicks
- Preserves natural speech rhythm
- Minimum 100ms gap between segments to maintain flow

**API Response Format**:

**Crop Calculation**:
```
Source: 1920x1080 (16:9)
Target: 9:16

1. Calculate target ratio: 9/16 = 0.5625
2. Source ratio: 1920/1080 = 1.78

3. Target is taller (0.5625 < 1.78) → Crop width
   crop_width = source_height * target_ratio = 1080 * 0.5625 = 607.5 → 608
   crop_height = source_height = 1080

4. Apply safe zone margin (10%):
   safe_width = 608 * 0.9 = 547

5. Max offset: 1920 - 608 = 1312 (can move crop window this far)
```

**Motion Smoothing Algorithm**:
```python
# Exponential moving average
smoothed_x = prev_x * factor + current_x * (1 - factor)
smoothed_y = prev_y * factor + current_y * (1 - factor)

# factor = 0.8 → smooth, slow response
# factor = 0.3 → responsive, some jitter
```

**Key findings**:
- FFmpeg crop filter syntax: `crop=w:h:x:y`
- Scale after crop for final dimensions
- setpts not needed for crop (no timestamp changes)
- Batch processing with semaphore prevents memory exhaustion

**Rep Counting Algorithm**:
```python
# Based on joint angle oscillation
1. Extract joint angles per frame (knee, hip, elbow)
2. Smooth with moving average (window=5)
3. Find zero-crossings around average
4. Count full cycles (descent + ascent)
```

**Form Analysis**:
- Asymmetry detection: Left/right angle differences >15°
- Depth scoring: Squat depth (knee angle <90° = good)
- Range of motion: Insufficient ROM flagged
- Category-specific tips from exercise library

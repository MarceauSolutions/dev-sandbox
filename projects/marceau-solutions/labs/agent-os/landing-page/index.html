Execution Discipline

12 Rules Forged By Failure

Every rule in AgentOS exists because something broke in production. These aren't opinions. They're battle scars turned into guard rails.

E1

Just Do It

Your AI stops asking permission for obvious next steps. If the file needs creating, it creates it. If the test needs running, it runs it. No "would you like me to?" for self-evident actions.

Origin: Counted 47 unnecessary permission requests in a single session. Every one broke flow state.
E10

Best-Path Evaluation

Before building anything, your AI answers 5 mandatory questions: Who's the user? What infrastructure fits? How does it scale? What are the real constraints? Is there a consolidation opportunity?

Origin: Claude repeatedly built CLI scripts when the user needed web dashboards. The fastest path to code is rarely the best path to use.
E11

Spec Written ≠ Shipped

A specification document is not a deliverable. A markdown plan is not a product. The pipeline is: Build, Deploy, Test on target, Verify user access. Only then mark complete.

Origin: An entire accountability system was "built" as markdown files but never deployed. The user woke up expecting it to work. It didn't.
E12

Code to Production Pipeline

CLI is never the final interface. Every tool goes through: interface decision, core logic, wire to interface, deploy to production, verify access, launch script, documentation.

Origin: Tools kept getting built as terminal scripts and declared "done." The user has to open VS Code to use a tool that should have been a Telegram command.

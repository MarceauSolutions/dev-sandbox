# Standard Operating Procedures — Index

> SOPs define how recurring tasks are done. Follow them exactly — they exist because ad-hoc approaches failed.

| SOP | Purpose | When to Use |
|-----|---------|-------------|
| [SOP 00](sop-00-kickoff.md) | Project Kickoff | Starting any new project |
| [SOP 01](sop-01-init.md) | Project Initialization | After kickoff, create directory structure |
| [SOP 03](sop-03-deployment.md) | Deployment | Deploying code to production |
| [SOP 07](sop-07-rollback.md) | Rollback | Reverting a bad deployment |
| [SOP 25](sop-25-documentation.md) | Documentation Decision | Deciding when/what to document |

## Adding New SOPs

1. Identify a recurring task that's been done inconsistently
2. Write the procedure in `docs/sops/sop-XX-name.md`
3. Add it to this index
4. Reference it in CLAUDE.md communication patterns if applicable

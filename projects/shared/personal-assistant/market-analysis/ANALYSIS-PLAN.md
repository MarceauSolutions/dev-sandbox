# Market Viability Analysis: DecideForHer

## Product Hypothesis

**What it is**: A mobile app that helps couples decide where/what to eat by tracking order history, learning preferences, and predicting what "she" might want based on patterns (including cycle timing).

**Who it's for**: Boyfriends/partners tired of the "where do you want to eat?" conversation

**Problem it solves**: Decision fatigue in couples around food choices, the stereotypical "I don't know, you pick" scenario

**How we'd make money**:
- Freemium with $4.99/mo premium
- Restaurant affiliate commissions
- Promoted restaurant placements

## Unique Value Proposition
"Stop asking. Start ordering."

## Key Features
1. Order history tracking per restaurant
2. Preference learning (cuisine, price, time of day)
3. Cycle-aware predictions (optional, privacy-first)
4. One-tap reorder from partner's phone
5. "She might want" predictions based on patterns
6. **Science-based cravings**: Sweet/savory/salty recommendations based on hormone levels

## Partnership Opportunities
- **Period tracking apps**: Flo, Clue, Natural Cycles - API integration for cycle data
- **Couples apps**: Paired, Honeydue - cross-promotion
- **Food delivery**: DoorDash, Uber Eats - affiliate + deep linking
- Could position as the "food layer" for existing relationship/health apps

DEPLOYMENT WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

Local Machine:
1. scp execution/mem0_api.py ec2:/home/ubuntu/dev-sandbox/execution/
2. scp execution/mem0-api.service ec2:/home/ubuntu/dev-sandbox/execution/
3. scp execution/install_mem0_ec2.sh ec2:/home/ubuntu/dev-sandbox/execution/
4. ssh ec2 'bash /home/ubuntu/dev-sandbox/execution/install_mem0_ec2.sh'

EC2 (during installation):
1. pip install fastapi uvicorn mem0ai python-dotenv
2. mkdir -p ~/.mem0/chroma_db
3. Test API startup
4. cp mem0-api.service /etc/systemd/system/
5. systemctl enable mem0-api
6. systemctl start mem0-api

Verification:
1. ssh ec2 'curl http://localhost:5020/health'
2. ssh ec2 'cd dev-sandbox && python3 -m execution.test_mem0_api'

TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

Service won't start:
  1. sudo journalctl -u mem0-api -n 50
  2. grep OPENAI_API_KEY /home/ubuntu/dev-sandbox/.env
  3. python3 -m pip list | grep mem0
  
Health check fails:
  1. sudo systemctl status mem0-api
  2. sudo lsof -i :5020
  3. curl -v http://localhost:5020/health
  
Out of memory:
  1. systemctl status mem0-api | grep Memory
  2. sudo systemctl restart mem0-api
  
Reset database:
  1. sudo systemctl stop mem0-api
  2. rm -rf ~/.mem0/chroma_db && mkdir -p ~/.mem0/chroma_db
  3. sudo systemctl start mem0-api
